# Documentation Guide

This folder holds supporting project documents so the repo root stays focused on the runnable app.

## Contents

- `BRIEF_AUDIT.md`: requirement-by-requirement audit against the challenge brief
- `PROJECT_INTERPRETATION.md`: baseline interpretation and scope decisions
- `process_log.md`: declared tool usage and build timeline
- `footprint_report.md`: package-size notes and measurement commands
- `SIGNED.md`: honor-code declaration
- `analysis/kt_eval.ipynb`: evaluation notebook kept out of the repo root

## Where to start

If you want to run the app, go back to the repo root and open `README.md`.

Main entrypoints remain:

- `app.py` for the Hugging Face Space
- `demo.py` for local Gradio runs
- `parent_report.py` for the standalone parent report
